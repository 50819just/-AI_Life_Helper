# FOOTER

## 文件用途

定義 AI Life Assistant 的 Footer 使用規則。

## Footer 定位

Footer 是輔助資訊區，可有可無。它不是 Home 或 module 的核心內容，也不應承擔主要任務 CTA。

本專案的 Footer 預設應非常簡約，不做大型網站 footer，也不放大量 navigation links。

Default footer style:

```txt
© AI Life Assistant
Demo Prototype
```

或：

```txt
AI Life Assistant · Product-like Demo Prototype
```

## Footer 可以包含

- Product / brand note
- Product version / demo label
- 最多 1–2 個 secondary links，例如 Help 或 Privacy

## Footer 不應包含

- 重要任務 CTA
- 三個正式 demo modules 的主要入口
- Coming Soon modules 的完整功能導流
- 大量 marketing links
- 取代 Header / Nav / Mobile Bottom Nav 的導航
- 重複 sidebar 裡已經有的主要 nav items

## 使用建議

### Desktop

可以有非常輕量 footer，但保持低視覺權重。

### Tablet

視內容長度決定是否保留 footer。

### Mobile

若頁面已有 fixed bottom navigation，footer 應非常簡短，避免與 bottom nav 競爭。

## Reference Screen Adjustment

如果 reference screen 的 footer 出現品牌文案加多個 footer nav links，例如：

```txt
靈動生活
Home / Dashboard / AI Chat / Settings
```

在本專案中應簡化為：

```txt
AI Life Assistant · Demo Prototype
```

不要在 footer 重複主要導航，也不要把 Dashboard / Settings 放成 footer 主要入口。

## Stitch 生成注意

- Footer 不要搶主要內容。
- 不要把 Coming Soon modules 放成完整 footer navigation。
- 若畫面已經很長，footer 可以省略。
- Footer 優先簡約，最多一行文字或一行文字加 1–2 個 secondary links。
